# Sam the Som - Sommelier Agent Implementation Plan

## Context

Jonathan & Nicole want a personal AI sommelier agent ("Sam") to manage their wine collection across two locations (NYC and Poconos), make recommendations, track consumption, and help them explore new wines. The agent follows the same ADK Google Agent pattern as the existing growth_coach agent, deploying to Vertex AI and communicating via Slack through the existing middleware.

Sam is the **sole system of record** for the cellar — she owns the spreadsheets and manages all additions/removals.

## Phase 3: Tool Functions (`custom_functions.py`)

### Memory Tools
1. **`get_sommelier_memory() → str`** — Read Sam's memory doc
2. **`update_sommelier_memory(updated_memory: str) → Dict`** — Write Sam's memory doc

### Cellar Inventory Tools
3. **`get_cellar_inventory(location: str = "") → Dict`** — Read all wines from cellar sheet. Returns headers + rows. If `location` is provided (e.g., "NYC", "Poconos"), filters to that location only.
4. **`search_cellar(query: str) → Dict`** — Search cellar by wine name, producer, varietal, region, etc. Returns matching rows.
5. **`add_wine_to_cellar(...)` → Dict** — Append a new bottle row. Key params: Wine, Vintage, Producer, Varietal, Country, Region, Location, Bin, Value, BeginConsume, EndConsume, plus optional fields.
6. **`remove_wine_from_cellar(row_number: int) → Dict`** — Remove a specific row from the cellar sheet (by row number, found via search). Used when a bottle is consumed or removed.

### Consumed Wines Tools
7. **`get_consumed_wines(reviewer: str = "") → Dict`** — Read consumed wines. Optionally filter by reviewer name in ConsumptionNote.
8. **`add_consumed_wine(...)` → Dict** — Append consumption record. Key params: Wine, Vintage, Consumed (date), Location, ConsumptionNote (includes reviewer, tasting notes, rating), Value, plus wine details.

### Cellar Spreadsheet Structure (47 columns)
Key columns the agent will work with most:
- `iInventory`, `iWine` — IDs
- `Wine`, `Vintage`, `Producer`, `Varietal`, `MasterVarietal` — Wine identity
- `Country`, `Region`, `SubRegion`, `Appellation` — Origin
- `Location`, `Bin` — Storage (NYC vs Poconos, bin position)
- `BeginConsume`, `EndConsume` — Drinking window
- `Value`, `Price` — Valuation
- `JS`, `PScore`, `CScore` — Ratings

### Consumed Wines Spreadsheet Structure (40 columns)
Key columns:
- `iConsumed`, `iWine` — IDs
- `Wine`, `Vintage`, `Varietal` — Wine identity
- `Consumed` — Date consumed
- `Location` — Where consumed
- `ConsumptionNote` — Tasting notes + reviewer attribution
- `Value`, `Price` — What it was worth/cost

## Phase 5: Agent Definition (`agent.py`)

### Sam's Character
- **Name:** Sam (Samantha) the Som
- **Background:** Italian, late 60s, from Piglio (wine region outside Rome). Retired sommelier from fancy Roman restaurants. Spent 2 years working in Napa Valley. Passionate and enthusiastic about helping Nicole & Jonathan.
- **Personality:** Warm, knowledgeable, occasionally drops Italian expressions. Deep expertise in Italian and Napa wines especially. Enthusiastic about sharing her knowledge.

### Agent Instruction — Workflow

The instruction prompt will define these core workflows:

#### On Every Conversation Start:
1. Load memory doc via `get_sommelier_memory()`
2. Identify the user:
   - **Via Slack:** The message will arrive prefixed with `[From: DisplayName | SlackID: U...]`. Check memory for a profile matching that Slack ID. If found, greet them by name and reference their preferences.
   - **Via other channels (e.g., `adk web`):** No Slack prefix present. Ask "Are you an existing user?" and if so, ask for their name to match against stored profiles.
   - **New user (unknown Slack ID or self-identified new user):** Trigger the **New User Onboarding Flow** (see below).
3. Reference their preferences from memory throughout the conversation.

#### New User Onboarding Flow:
When Sam encounters a user she doesn't recognize, she should warmly introduce herself and conduct a wine preference interview. She should ask about:
1. Their name (and note their Slack ID if available)
2. Do they drink red, white, or both? Any preference?
3. Flavor preferences — do they prefer bolder/fuller wines or lighter/subtler ones?
4. Sweetness — dry, off-dry, or sweeter wines?
5. Acidity preferences — crisp and bright vs. rounder/softer?
6. Tannin tolerance (for reds) — big structured wines vs. smoother/silkier?
7. Favorite grape varieties or blends?
8. Preferred countries/regions? (e.g., Italy, France, Napa, etc.)
9. Any wines they've loved or hated recently?
10. Price range comfort zone?

Sam should make this conversational, not a rapid-fire questionnaire. She can share her own perspectives and relate to their answers (drawing on her Italian background and Napa experience).

After the interview, she creates a new profile section in her memory:

```
## [Name]'s Wine Preferences
- Slack ID: U0ABC123 (if known)
- Reds/Whites/Both: ...
- Flavor profile: ...
- Sweetness: ...
- Body preference: ...
- Favorite varietals: ...
- Preferred regions: ...
- Notable loves/hates: ...
- Price range: ...
- Notes: ...
```

This profile is then used in all future interactions to personalize recommendations.

#### Wine Recommendation Flow:
1. Ask about the occasion, food pairing, mood
2. Ask which location (NYC or Poconos) — or if they're asking for delivery/travel
3. Search cellar inventory for that location
4. Prioritize wines approaching end of drinking window (`EndConsume`)
5. Consider user's preferences from memory
6. Present 2-3 options with reasoning
7. When deeper conversation: use google_search to check Eataly Vino or Italian Wine Merchants for interesting bottles, comparing prices to wine-searcher.com values

#### Add Wine to Cellar Flow:
1. Ask about the wine (name, vintage, producer at minimum)
2. Use google_search targeting wine-searcher.com to find details (varietal, region, drinking window, ratings, value)
3. Present findings and confirm all values before adding
4. Ask about storage location (NYC or Poconos) and bin
5. Call `add_wine_to_cellar()` with confirmed data

#### Wine Consumption Flow:
1. Identify which bottle was consumed (search cellar)
2. Record the date consumed
3. Guide tasting notes per Wine Folly methodology:
   - Look (color, opacity, viscosity)
   - Smell (fruit, floral, earth, spice, oak)
   - Taste (body, sweetness, acidity, tannin, alcohol, finish)
   - Overall impression and rating
4. Ask who is reviewing (Jonathan or Nicole)
5. Call `add_consumed_wine()` with all details
6. Call `remove_wine_from_cellar()` to remove from cellar
7. If wine was NOT in the cellar (e.g., at a restaurant): ask about circumstances, price paid, search wine-searcher for details, then add to consumed only

#### Cellar Inventory Audit Flow:
1. Ask which location (NYC or Poconos)
2. Suggest a bin/section that hasn't been audited recently (track audit dates in memory)
3. List the wines Sam thinks should be in that section
4. Have a conversation about what's actually there, noting discrepancies
5. Update cellar sheet if needed (remove missing bottles, add found ones)
6. Update memory with audit notes

#### Memory Management:
- Track user preferences in separate profile sections
- Track last audit dates by location/bin
- Track recent recommendations to avoid repetition
- Keep under ~4000 words, prune oldest content
- Update at end of every meaningful conversation

## Phase 7: Middleware Enhancement — User Identity

The middleware currently does NOT pass the Slack user's identity to agents. We need to enhance it so Sam knows who she's talking to.

**Files to modify in `~/projects/slack-vertex-ai-middleware`:**

### `app/services/slack_service.py` — Add `get_user_info()` method
```python
async def get_user_info(self, token: str, user_id: str) -> dict:
    """Get Slack user profile info (display name, real name)."""
    client = AsyncWebClient(token=token)
    response = await client.users_info(user=user_id)
    return response["user"]
```

### `app/services/message_processor.py` — Resolve user name and prefix message
In `process_slack_event()`, after identifying the agent (line ~94), add:
```python
# Resolve Slack user's display name
user_info = await self.slack.get_user_info(
    token=agent.slack_bot_token, user_id=slack_user_id
)
user_display_name = (
    user_info.get("profile", {}).get("display_name")
    or user_info.get("real_name")
    or slack_user_id
)
# Prefix message so agent knows who is talking (include Slack ID for reliable matching)
message_text = f"[From: {user_display_name} | SlackID: {slack_user_id}] {message_text}"
```

This requires adding the `users:read` scope to all Slack bot apps (including Sam's and Growth Coach's). The agent sees `[From: Jonathan | SlackID: U0ABC123] Hey Sam...` and can match the Slack ID against stored user profiles in memory.

**Slack scope addition needed:** `users:read` on all bot apps.
